# AI-pets

> 一个具有智能交互能力的桌面宠物应用 | [MVP 验证阶段]

## 项目简介

本项目是一款基于 Python + PySide6 开发的 AI 桌面宠物应用，旨在为用户提供一个可爱、智能化且具有陪伴属性的桌面助手。核心价值在于将传统桌面宠物的娱乐功能与 AI 智能交互、使用行为分析相结合，打造一个既能陪伴用户又能感知用户使用习惯的智能桌面伴侣。

当前项目已成功验证核心产品思路和关键技术链路的可行性，所有核心功能均已在 MVP 阶段完成技术验证。

## 核心痛点与解决方案

### 痛点 1：传统桌面宠物的"僵硬感"

**问题描述**：市面上的桌面宠物大多只有简单的预设动画，交互模式固定，用户很快就会失去新鲜感。

**解决方案**：本项目创新性地将 AI 对话能力融入桌面宠物交互中。通过集成大语言模型 API，用户可以与宠物进行自然语言对话，获得情感化的交互体验。技术实现上采用后台线程处理 AI 请求，确保 UI 线程始终保持流畅，不会因为 AI 响应延迟而影响宠物动画的流畅度。

### 痛点 2：用户对应用使用行为缺乏感知

**问题描述**：大多数用户并不清楚自己每天在不同应用上花费了多少时间，缺乏对数字生活方式的认知和管理工具。

**解决方案**：项目实现了完整的 Windows 进程监控和数据分析系统。核心技术挑战在于准确识别用户应用（而非系统进程），通过多维度过滤策略（进程路径、名称、父进程、权限检查）结合 Windows 注册表数据匹配，成功构建了可靠的应用使用时间追踪能力。

## 技术架构与选型

### 核心技术栈

| 技术选型 | 版本要求 | 用途说明 |
|---------|---------|---------|
| **Python** | 3.12+ | 主开发语言，现代 Python 特性支持 |
| **PySide6** | >= 6.0.0 | Qt for Python，跨平台 GUI 框架 |
| **psutil** | >= 5.8.0 | 进程和系统监控 |
| **requests** | >= 2.28.0 | HTTP 客户端，AI API 调用 |
| **winreg** | 内置 | Windows 注册表访问 |
| **threading** | 内置 | 后台任务处理 |



## MVP 验证成果

### ✅ 核心功能验证成功

- **流畅的交互体验**：拖拽移动、桌面置顶、右键菜单等交互功能均已稳定运行
- **AI 对话能力**：成功集成了通用大语言模型 API，实现了上下文连续的对话功能，异步调用机制验证有效
- **使用时间统计**：成功构建了 Windows 进程监控闭环，从进程识别→时间计算→数据持久化→可视化展示全链路验证通过
- **系统进程过滤**：通过多层过滤策略（路径白名单、关键词匹配、父进程分析、权限检查）有效过滤系统进程，验证了技术可行性

### ✅ 技术能力验证成功

- 透明无边框窗口的实现方案
- 基于 QThread 的异步任务处理架构
- Windows 注册表与进程信息的联合查询
- 多进程 CPU 时间的精确计量
- 配置数据的持久化方案（QSettings + JSON）
